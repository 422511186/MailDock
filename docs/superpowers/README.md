# Superpowers 文档说明

本目录保存项目建设过程中产生的历史设计稿和实施计划。它们适合用来理解当时为什么做出某些取舍，但不保证与当前代码完全一致。

查看当前架构和开发约束时，请优先参考这些资料：

- `README.md`：面向使用者的项目概览、启动方式、API 列表和当前高层架构。
- `AGENTS.md`：编码代理使用的开发约束、架构边界、测试策略和当前前后端职责说明。
- `CLAUDE.md`：面向 Claude Code 的当前开发指南，内容与 `AGENTS.md` 保持同一事实基线。
- 源码和测试：当文档与代码不一致时，以源码和测试为最终事实来源。

`specs/` 和 `plans/` 下的文件请按历史记录阅读。部分旧文档可能仍提到已经过时的细节，例如 `WebVerticle` 负责依赖装配、`ApiRouter` 内联具体 handler，或 `App.tsx` 使用手写视图状态机而不是 `react-router`。这些内容描述的是文档创建时的项目状态，不一定代表当前实现。

后续新增架构说明时，优先更新上面列出的当前指南文件；只有在需要记录新的设计提案或实施路径时，再新增带日期的 spec / plan。
